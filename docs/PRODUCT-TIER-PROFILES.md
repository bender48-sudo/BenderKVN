# Три уровня профиля в подписке (по референсу WL/VPN)

**Источник:** разбор рабочих конфигов конкурента (**2026‑05‑18**). UUID/ключи в репо **не** храним — только паттерны.  
**Задача в бэклоге:** **P1-PRO-SUB-TIER-01** · **Q062**.

---

## Идея продукта

Одна подписка — **несколько outbounds** с разными ролями. Пользователь (или саппорт) выбирает узел под ситуацию:

| Уровень | Имя (рабочее) | Когда | Скорость | Сложность |
|---------|---------------|-------|----------|-----------|
| **1** | **turbo** / `nl-public` | Обычный интернет, Wi‑Fi/LTE **без** усечённого WL | Максимум | Низкая |
| **2** | **wl-direct** | WL на мобиле/роутере **с** своей маршрутизацией; RU-сервисы «не важны» | Средняя | Средняя |
| **3** | **wl-routed** | WL + нужны **все** RU-приложения без «VPN включён» | Ниже, выше ping | Высокая |

«Чем круче обход WL — тем медленнее» — закладываем в UX (подписи в Happ / portal).

---

## Паттерны из референса → BenderVPN

| Паттерн | Референс | У нас сейчас | Действие |
|---------|----------|--------------|----------|
| Зарубежная нода «как есть» | NL, **без** routing rules | **NL:9443**, LV:443, MUX ✅ | Оставить **tier-1**; имя в sub «BenderVPN · NL» |
| WL: прямой туннель | RF IP (**158.160.x** ≈ Yandex Cloud), **:80**, Reality, **sni=www.yandex.ru**, **без** `flow` | RU-bypass **на сервере** ✅; SNI **github/bing** на LV | **P4-DNS-07** RF egress; **P2-RED-SNI-ROTATE-01** → **yandex.ru**; inbound **:80** |
| WL: маршрутизированный | Тот же RF, **:443**, **`flow=xtls-rprx-vision`**, routing на клиенте | Частично **RU-BYPASS** в шаблоне; нет отдельного «mobile routed» | **tier-3** outbound + правила; **только iOS/Android** в FAQ |
| Fingerprint | `fp=chrome` | Зависит от шаблона | Явно **chrome** в матрице |
| Порт **80** для Reality | Оба WL | Только 443/9443 | Alt inbound **80** на RF-ноде (Q062) |
| Роутер | «Подключите к роутеру — прибью» | Не сказано | **ONBOARDING/FAQ:** tier-3 **не** для Keenetic/OpenWrt |

## Целевая матрица (после Q062)

| Tier | Нода | Порт | SNI (Reality) | flow | Routing |
|------|------|------|---------------|------|---------|
| **1 turbo** | **NL** (и LV primary) | **443** / **9443** | уйти с github → **yandex.ru** или neutral | vision по политикике ISP | минимум (MUX) |
| **2 wl-direct** | **RF egress** (новая) | **80** | **www.yandex.ru** | **нет** | **нет** (немаршрутизированный) |
| **3 wl-routed** | **RF egress** | **443** | **www.yandex.ru** | **xtls-rprx-vision** | **да** (geosite:ru direct + tunnel rest) |

**Не смешивать** tier-3 с **роутером** — только телефон (Happ split / будущий native app).
