## Где в бэклоге

- **Исполнение:** линейная очередь **`docs/BACKLOG-QUEUE.md`** (**одна** строка **`NEXT`** → коммит → новая сессия). Правило: **`docs/POLICY-SEQUENTIAL-WORK.md`**.

- Спринты и группы задач: **`docs/COMMERCIAL-BACKLOG.md` §3** (карта, не порядок работы).
- Пользовательский флоу (бабушка-тест, сайт без VPN): **`docs/USER-FLOW-BACKLOG.md`** → **§7.1** коммерческого бэклога.

- Закрытие **P2-OPS-DRIFT-POST-P0**: см. **`docs/DRIFT-POST-P0.md`**.

## Кратко

**Качественный продукт** = работающий VPN + панель + понятные инциденты для операторов.  
**Удобство пользователя** = следующий слой, когда инженерный контур не отваливается из‑за долга по репо и метрикам.
