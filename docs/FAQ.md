# BenderVPN — FAQ для пользователя (подключение, оплата, поддержка)

Цель документа — **P1-PRO-01**: однозначная линия продукта, чтобы не смешивать ожидания
«основной VPN» и дополнительных режимов.

**Данные:** какие поля собираем и чего не храним — **`docs/DATA-MINIMIZATION-POLICY.md`**
(полные юридические тексты — ссылки «Условия» / «Конфиденциальность» в боте).

## Что вы покупаете

**Основной продукт** — коммерческий **VPN на базе Xray/Reality/VLESS**, доступ по подписке
через панель. Трафик идёт через **production-ноды** провайдера; часть ресурсов
(в т.ч. российские приложения и сайты по политике **split‑tunnel / RU‑bypass**)
может направляться **мимо VPN (direct)** — это описано на стороне шаблона подписки,
а не означает «VPN выключен» для всего интернета.

Подключение выполняется в клиенте на **каждом устройстве**, где нужен VPN:

| Устройство | Рекомендуемый клиент |
|------------|----------------------|
| iPhone / iPad | **Happ** (App Store) |
| Android | **Happ** (Google Play и др.) |
| **Windows** (ПК, ноутбук) | **Happ** для Windows |
| **Mac** | **Happ** (Mac App Store) |

**Happ** есть и **на телефоне**, и **на компьютере** — это одно приложение, но его нужно **установить на каждое устройство отдельно** и добавить туда вашу ссылку настройки.

Также возможны **Streisand**, **Hiddify**, **v2rayN** и др.; поддерживаемые комбинации зависят от клиента и актуальной выдачи подписки.

## Чего этот SKU **не** является

**Аварийный / DNS‑режим**, «белые списки», bootstrap только через DNS (например
отдельный PoC **`P4-DNS`** в бэклоге) — **не** замена полноценному VPN и **не**
обещается как основной канал доступа во всё приложение, если это явно не вынесено
в отдельную оферту/страницу продукта.

**Bootstrap по DNS ≠ «весь интернет через VPN».** DNS может помочь **первым шагом**
(узнать, что сервис жив, резолвить имя или получить служебную подсказку), но
**политика SKU BenderVPN** остаётся: защита трафика — через **VPN‑подписку и клиент**.
Не обещать пользователю, что один DNS решит блокировку всего приложения без туннеля.

Если у вас включён только DNS без полноценного VPN‑туннеля — вы **не** в том сценарии,
который описывает данный FAQ по умолчанию.

## Локации и «падения» одной ноды

Провайдер стремится к **минимизации простоя** при деградации одной ноды или панели
поддержки клиентским потокам. При инциденте смотрите **публичный статус**: `https://k9x2m1.conntest.xyz:8443/status`
(обновляется каждые ~2 мин). Поддержка — через Telegram-бот; внутренние шаги — **`docs/RUNBOOK-INCIDENT.md`**.

## Пробный период для новых подключений

С **2026‑05‑16** для **новых** пользователей, которые впервые получают ключ через бота,
заложено **до ~90 календарных дней** бесплатного доступа («три месяца» в смысле продуктовой политики;
точную дату окончания показывает панель и карточка ключа в боте).

Пользователи, уже учтённые на панели **до этой политики**, получают **долгий срок действия профиля** (по умолчанию до **2099** — разовая процедура в **`docs/DEPLOY.md`**). Это нужно, чтобы не «обрезать» текущую базу при смене правил пробного доступа.

**Окончание срока:** бот отправляет напоминания заранее (за несколько дней и в день окончания).

## Обновление подписки

После изменений на стороне сервера (маршрутизация, ноды) может потребоваться
**обновить профиль подписки** в приложении (кнопка обновления / pull subscription),
затем переподключиться. Это типовой ответ поддержки при «ошибках Xray» после
глобальных правок конфигурации.
