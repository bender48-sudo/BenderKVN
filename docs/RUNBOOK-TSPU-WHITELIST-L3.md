# Runbook: режим «белый список» L3/L7 (мобильный интернет, 2026)

**ID:** **P2-RED-WHITELIST-L3-01** (Q105).  
**Контекст:** [Habr — белые списки](https://habr.com/ru/articles/1027276/), реестр ~75k IP ([Коммерсант](https://www.kommersant.ru/doc/7659315)).

## Симптомы

- LTE: **не открывается** portal `k9x2m1` / `p4n7q` (**:8443**), Telegram иногда жив.
- Wi‑Fi дома: всё OK.
- `tspu_block_probe_ru.py` с relay: edge **timeout**, хотя VPN «вчера работал».

## Механика (кратко)

| Уровень | Что режет |
|---------|-----------|
| **L3** | IP VPS **вне** корпоративного whitelist |
| **L7** | SNI в чёрном списке даже при «разрешённом» IP |

BenderVPN **не в** корпоративном реестре → при ужесточении режима **NL/LV IP** могут быть недоступны для bootstrap.

## Действия support

1. Не обещать «обнови подписку» если **portal не грузится** — сначала канал доставки ссылки (бот, зеркало TG).
2. Проверить **`/status`** с Wi‑Fi vs LTE (скрин + оператор).
3. Если включён только DNS-PoC (**P4-DNS**) — напомнить: **DNS ≠ полный VPN** ([`FAQ.md`](FAQ.md)).
4. Эскалация ops: RF egress (**Q060**), backup bootstrap domain (**Q047**).

## Ops

- Мониторинг: отдельный тикет «WL-mode» в incidents.json (ручной флаг владельца).
- Не менять SNI/порт всей базе из‑за WL — это **другой** класс инцидента.
