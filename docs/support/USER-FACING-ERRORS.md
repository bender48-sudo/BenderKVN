# Сообщения для пользователя и поддержки (P3‑UX‑02)

Карта: **что видит поддержка в логах** → **что говорить пользователю** (без внутренних кодов).

| Симптом в логе / мониторинге | Пользователь (RU) |
|------------------------------|-------------------|
| Timeout / ConnectError при вызове API панели | «Сервис на секунду не ответил. Проверьте интернет и повторите. Если долго не проходит — напишите в поддержку». |
| HTTP 401/403 от панели при операциях бота | «На стороне сервера обновляется авторизация. Попробуйте через несколько минут; если ошибка сохраняется — поддержка». (Операторам: см. **`docs/RUNBOOK-REMNA-API-TOKEN.md`**, **`JWT_API_TOKENS_SECRET`**) |
| Prisma / 502 subscription-page при обновлении ссылки подписки | «Подождите немного и нажми „обновить подписку“ в приложении ещё раз». (Инцидент: см. **`docs/RUNBOOK-INCIDENT.md`**) |
| TLS certificate verify failed из скриптов | Для пользователя: «У вас может быть блокировка провайдером». Для поддержки не говорить сырой `CERTIFICATE_VERIFY_FAILED`; проверять дату клиента/VPN-профиль (см. **`docs/FAQ.md`**). |
| «message is not modified» (Telegram) | Это не ошибка UX; можно игнорировать. |

Константы для бота импортируются как **`from shop_bot.bot import user_messages`** (файл в репо — **`bot_src/user_messages.py`**; при деплее он лежит рядом с **`handlers`** в **`shop_bot/bot/`**).

Расширять таблицу при новых паттернах логов.
