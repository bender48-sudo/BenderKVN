# Политика SNI‑проб и мониторинга (снижение сигнатуры)

**P1-PRO-04**. Внутренний документ для команды; не публикуется пользователям как юридический текст.

## Зачем мы делаем SNI‑пробы

- **RU‑relay зона** (`ru-monitor.py`): проверка доступности выбранных **SNI** через российский relay — это **операционный надзор** за цепочкой «клиент ↔ нода», а не разведка пользовательского трафика.
- **Selfsteal / Caddy** (`selfsteal-monitor.py`): проверка ответов по набору SNI на **своих** edge‑хостах (ожидаемые коды, tolerances).

## Принципы минимизации сигнатуры

1. **Только хосты из панели** — список целей для `ru-monitor` строится из API (`hosts`), с фильтрами **`isDisabled`** / скрытие, без ручного «расширять и забыть».
2. **Не добавлять лишние публичные домены** в мониторинг без причины (каждый новый SNI — обоснование в комментарии к PR/задаче или в этом файле приложением строки ниже).
3. **Разделение**：юзерский VPN-путь ≠ обязательно тот же набор проверок, что и маркетинговые домены для selfsteal; не смешивать цели без архитектурной необходимости.
4. **Частота** — задаётся cron на LV/AMS; изменения частоты согласовывать (риск нагрузки на relay и внешние точки).
