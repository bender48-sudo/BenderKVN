# Политика логов и данных (черновик P3‑TR‑01)

Внутренний документ: **не** публичная политика; согласовать с офертой перед публикацией.

## Принципы

1. **Минимум ПДн в Telegram-алертах** — см. **`docs/POLICY-TELEGRAM-ALERTS.md`**.
2. **Подписка `/api/sub/*`** на edge — доступ-логи без сырого пути там, где включён **`log_skip`** — см. **`docs/RUNBOOK-CADDY-SUBSCRIPTION-LOGS.md`**.
3. **Панель Remnawave** — логи приложения и Postgres на AMS; доступ только root/admin с SSH; ротация по диску провайдера.
4. **Бот** — приложение-пишущее в stdout контейнера; не пересылать полные дампы пользователю без вырезания токенов.
5. **Бэкапы БД панели** — **`docs/RUNBOOK-BACKUP-REMNAWAVE.md`**; локальное хранилище чувствительно.

## Retention (ориентир до отдельного решения)

| Источник | Ориентир | Примечание |
|----------|----------|------------|
| Telegram alert history | бессрочно в TG (политика канала) | метаданные минимизировать |
| `monitor.sh` / `ru-monitor.py` logs на LV | ротация logrotate если настроено | см. **`DEPLOY.md`** crontab |
| SQLite бота на AMS | см. операционную практику бэкапов бота отдельно от панели | не в этом репо |
