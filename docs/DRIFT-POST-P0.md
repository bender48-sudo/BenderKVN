# Drift прод ↔ репо после P0-SEC-05 / смены панели (P2-OPS-DRIFT-POST-P0)

Задача бэклога: **`python ops/drift-check.py`** завершается с **exit 0** по всем парам из **`ops/drift-check.py` → `PAIRS`**, **или** по каждому оставшемуся расхождению заполнена строка в **таблице waive** ниже с **датой ревью** и понятным «когда снять».

**Неправильно:** пустая waive при наличии DRIFT в последнем прогоне и отсутствии работ по выравниванию — это не закрытие задачи, а долг.

## Как прогонять

1. Рабочая машина с настроенным **`~/.ssh/config`** (**`bvpn-lv`**, **`bvpn-ams`**, **`bvpn-nl`**) и доступом к репо.
2. Заполнен **`.secrets/vault.env`** (для **tmpl**-пар). См. **`docs/DEPLOY.md` §7**.
3. Из корня репозитория:

   ```bash
   PYTHONUNBUFFERED=1 python ops/drift-check.py
   ```

4. Таймауты на LV — возможны; скрипт делает **ретраи** (**`P2-ENG-DRIFT-CHECK-01`**). При стабильном **TIMEOUT** — повторить с более тихого канала или с хоста ближе к сети серверов.

**Exit codes:** **0** — все **OK**; **1** — DRIFT / TIMEOUT / проблемы с vault. На Windows оболочка иногда показывает ненулевой код как большое число — смотреть текст отчёta (**`problems:`**).

## Интерпретация

- **OK** — по этой паре ничего не делаем.
- **DRIFT (file)** — по умолчанию **SoT = репозиторий**: бекап на проде → выкладка файла из репо по **`docs/DEPLOY.md` §3** (не править «на живую» в `/opt/scripts/`). Исключение — горячий патч на проде: тогда сначала **`scp` с прода в `.secrets/prod-pull/`**, сравнение с репо, перенос правки в git, потом деплой канонического файла.
- **DRIFT (tmpl)** — сравнивается **рендер шаблона из vault** с продом. Значит: либо **vault устарел** относительно прода, либо **шаблон в репо** ушёл вперёд/в сторону, либо на проде руками правили compose/env. Порядок: **`docs/DEPLOY.md` §7.3** (sanitize / extract-vault / рендер / накат / при необходимости **`docker compose`**).

## Устранение drift (правильный порядок)

Делать **сверху вниз**; после каждого крупного блока — повторный **`python ops/drift-check.py`**.

1. **Vault**  
   Убедиться, что **`.secrets/vault.env`** отражает актуальные секреты для рендера. Если правда на проде после ротации — **`python .secrets/extract-vault.py`** (и при необходимости **`sanitize-compose`**) по **`docs/DEPLOY.md` §7.3**, не подгоняйте vault «на глаз».  
   Особенно: **`DATABASE_URL`** / **`POSTGRES_PASSWORD`** для **`remnawave-db`** должны совпадать с **инициализированным** томом Postgres (после **P0-SEC-05** — не подставляйте старый пароль из шаблона). Иначе панель уйдёт в crash loop (**Prisma P1000**).

2. **Скрипты `kind=file` (без compose)**  
   С **LV**: `balancer.sh`, `backup-remnawave.sh`, `ru-monitor.py`; с **LV + AMS**: `deploy-node.sh` — деплой из репо с **`bash -n` / `ast.parse`** и бекапом, см. **`docs/DEPLOY.md` §3**.  
   Готовые обёртки (если покрывают файл): `ops/deploy-ru-monitor-lv.ps1`, и т.д.; иначе универсальный **`scp` + `install`**.

3. **Compose и `.env` (`kind=tmpl`)**  
   Для каждой строки DRIFT смотреть **`tmpl_only_keys`** в **`ops/drift-check.py`** и рендерить так же, как в проверке: **`docs/DEPLOY.md` §7.2**.  
   На **AMS** после замены файлов в `/opt/remnawave/`, `/opt/remnawave/sub/`, `/opt/remna-shop/` — осознанно **`docker compose up -d`** / **`--force-recreate`** по **`docs/SECRETS.md`** и runbook’ам, без простоя там, где критично.

4. **Финиш**  
   **`python ops/drift-check.py`** → **exit 0**. Строка в **`docs/COMMERCIAL-BACKLOG.md` §12** (дата, что выровняли).

## Таблица waive (только намеренное временное расхождение)

Заполнять **только** если расхождение осознанное и согласовано (например, горячий патч на проде до merge, A/B эксперимент с датой окончания).
