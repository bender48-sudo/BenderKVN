# Сниппет для Cursor → User Rules (глобально)

Официально **User Rules** настраиваются в **Cursor Settings → Rules** (текст вручную). Дубликат для буфера — также в профиле: `%USERPROFILE%\.cursor\RULE-PASTE-INTO-USER-RULES.md`.

Скопируйте в **User Rules** только текст между линиями:

---

После **заметного** изменения кода или ops-скриптов делай **git commit и push в `main`** без отдельной просьбы, но **только после диагностики**, что всё работает:

- **Shell:** `bash -n` для изменённых `.sh` (на целевом хосте или по SSH).
- **Python:** `python -m py_compile` или тесты/линтер проекта.
- Если затронут прод — по возможности **smoke** на LV/AMS (команда завершается с exit 0).

Если проверка не прошла — **не коммить** до исправления. В сообщении коммита кратко укажи *зачем* меняли.

---

См. также `.cursor/rules/commit-after-verify.mdc` в этом репозитории (project rule).
