# Своё приложение BenderVPN (P5-PROD-NATIVE-APP-01)

**Статус:** product brief / R&D (очередь **Q053**, не блокирует **Q051–052**).  
**Задача в бэклоге:** **`docs/COMMERCIAL-BACKLOG.md` §9**.

## Зачем

- **Happ** — единственный практичный клиент в **App Store** в РФ; зарубежные (Streisand, Hiddify и т.п.) для массы недоступны.
- Фидбек бета (**2026‑05‑18**): недоверие к Happ как к РФ‑продукту; ограничения **split routing** на iOS (**Network Extension ~15 MB**).
- Долгосрочно: свой клиент с **GeoIP на устройстве** (трафик в РФ — direct, остальное — туннель), чтобы убрать «баннеры 1984» и не зависеть от чужого store‑клиента.

## MVP (черновик scope)

| Платформа | MVP | Не в MVP v0 |
|-----------|-----|-------------|
| **iOS** | VLESS/Reality из подписки; обновление sub; базовый connect/disconnect | Полные «пиздатые списки» на 15 MB; обход всех DPI-сценариев |
| **Android** | То же + проще с memory | — |
| **Общее** | Импорт `subscriptionUrl` как у Happ; статус/ошибки по **`USER-FACING-ERRORS`** | Свой биллинг (оплата остаётся в боте) |

## Архитектурные решения (Q053, черновик для владельца)

1. **Ядро:** **sing-box** mobile fork (как у конкурентов на NE); оценка **6–9 мес** до TestFlight при 1 FTE.
2. **Routing:** **клиентский GeoIP** для tier-3; серверный **RU-BYPASS** остаётся для Happ/legacy.
3. **App Store:** go/no-go **от владельца**; до решения — **Android first** (без NE лимита 15MB).
4. **Сосуществование:** Happ + v2rayN (**Q052**) до native; URL подписки **:8443** (**Q051** ✅).
